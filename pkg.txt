# Creating and Deploying a Flutter Package: Step-by-Step Guide

This comprehensive guide walks you through creating, testing, and publishing a Flutter package to pub.dev, using your gradient text widget as a practical example.

## Prerequisites

Before starting, ensure you have:

\begin{itemize}
\item Flutter SDK installed (check with \texttt{flutter --version})
\item A Google account for pub.dev authentication
\item Git installed for version control
\item A GitHub account (recommended for hosting the repository)
\item Your package code ready (gradient text widget in this case)
\end{itemize}

## Step 1: Create the Package Structure

Create a new Flutter package using the Flutter CLI:

flutter create --template=package gradient_text
cd gradient_text

This generates the following structure[1]:

gradient_text/
├── lib/
│   └── gradient_text.dart
├── test/
│   └── gradient_text_test.dart
├── example/
├── pubspec.yaml
├── README.md
├── CHANGELOG.md
└── LICENSE

**What each directory does:**
- `lib/` - Contains your package source code
- `test/` - Unit and widget tests for your package
- `example/` - Example app demonstrating package usage
- `pubspec.yaml` - Package metadata and dependencies
- `README.md` - Package documentation
- `CHANGELOG.md` - Version history
- `LICENSE` - License information (typically BSD, MIT, or Apache)

## Step 2: Organize Your Package Files

Move your gradient text files into the proper package structure:

### Main Library File

Create `lib/gradient_text.dart` as the main entry point:

library gradient_text;

export 'src/gradient_text_widget.dart';
export 'src/constants.dart';

### Source Files

Create the `lib/src/` directory for implementation files:

lib/
├── gradient_text.dart (main export file)
└── src/
    ├── gradient_text_widget.dart (your gradient-2.dart content)
    └── constants.dart (your constants.dart content)

**Update import statements** in `gradient_text_widget.dart`:

import 'package:flutter/material.dart';
import 'constants.dart';  // Local import, not package import

**Why this structure?** The `src/` directory is private by convention. Users import only `package:gradient_text/gradient_text.dart`, and you control what's exported[1][2].

Edit `pubspec.yaml` with your package metadata[2][3]:

  - widget

## Step 4: Write Documentation

### README.md

Create comprehensive documentation in `README.md`[2]:

# Gradient Text

A Flutter package for creating beautiful gradient text with customizable directions and types.

## Features

- Linear and radial gradients
- 8 gradient directions (LTR, RTL, TTB, BTT, and diagonal)
- Customizable colors and gradient stops
- Full TextStyle support
- Works with all Text widget properties

## Installation

Add to your `pubspec.yaml`:

dependencies:
  gradient_text: ^0.1.0

Then run:

flutter pub get

## Usage

### Basic Linear Gradient

import 'package:gradient_text/gradient_text.dart';

GradientText(
  'Hello World',
  colors: [Colors.blue, Colors.purple],
  gradientDirection: GradientDirection.ltr,
  gradientType: GradientType.linear,
  style: TextStyle(fontSize: 40, fontWeight: FontWeight.bold),
)

### Radial Gradient

GradientText(
  'Radial Text',
  colors: [Colors.red, Colors.orange, Colors.yellow],
  gradientType: GradientType.radial,
  radius: 1.5,
)

## Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| text | String | required | The text to display |
| colors | List<Color> | required | Gradient colors (min 2) |
| gradientDirection | GradientDirection? | ltr | Direction for linear gradient |
| gradientType | GradientType | linear | Linear or radial gradient |
| style | TextStyle? | null | Text styling |

## Example

See the [example](example/lib/main.dart) directory for a complete sample app.

## Step 5: Create an Example App

Create `example/lib/main.dart` to demonstrate your package:

import 'package:flutter/material.dart';
import 'package:gradient_text/gradient_text.dart';

void main() => runApp(const MyApp());

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Gradient Text Example',
      home: Scaffold(
        appBar: AppBar(title: const Text('Gradient Text Examples')),
        body: Center(
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              GradientText(
                'Linear Gradient',
                colors: const [Colors.blue, Colors.purple],
                gradientType: GradientType.linear,
                style: const TextStyle(fontSize: 40, fontWeight: FontWeight.bold),
              ),
              const SizedBox(height: 20),
              GradientText(
                'Radial Gradient',
                colors: const [Colors.red, Colors.orange, Colors.yellow],
                gradientType: GradientType.radial,
                style: const TextStyle(fontSize: 40, fontWeight: FontWeight.bold),
              ),
              const SizedBox(height: 20),
              GradientText(
                'Top to Bottom',
                colors: const [Colors.green, Colors.teal],
                gradientDirection: GradientDirection.ttb,
                style: const TextStyle(fontSize: 30),
              ),
            ],
          ),
        ),
      ),
    );
  }
}

## Step 6: Write Tests

Create comprehensive tests in `test/gradient_text_test.dart`:

import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:gradient_text/gradient_text.dart';

void main() {
  group('GradientText', () {
    testWidgets('renders with required parameters', (tester) async {
      await tester.pumpWidget(
        const MaterialApp(
          home: Scaffold(
            body: GradientText(
              'Test',
              colors: [Colors.blue, Colors.red],
            ),
          ),
        ),
      );

      expect(find.text('Test'), findsOneWidget);
    });

    testWidgets('applies linear gradient', (tester) async {
      await tester.pumpWidget(
        const MaterialApp(
          home: Scaffold(
            body: GradientText(
              'Linear',
              colors: [Colors.blue, Colors.red],
              gradientType: GradientType.linear,
            ),
          ),
        ),
      );

      expect(find.byType(ShaderMask), findsOneWidget);
      expect(find.text('Linear'), findsOneWidget);
    });

    testWidgets('applies radial gradient', (tester) async {
      await tester.pumpWidget(
        const MaterialApp(
          home: Scaffold(
            body: GradientText(
              'Radial',
              colors: [Colors.blue, Colors.red],
              gradientType: GradientType.radial,
            ),
          ),
        ),
      );

      expect(find.byType(ShaderMask), findsOneWidget);
    });

    test('asserts minimum two colors', () {
      expect(
        () => GradientText(
          'Test',
          colors: const [Colors.blue],
        ),
        throwsAssertionError,
      );
    });
  });
}

Run tests with:

flutter test

## Step 7: Validate Package Quality

Flutter provides tools to ensure your package meets pub.dev standards.

### Run Package Analyzer

Check for issues:

flutter analyze

Fix all errors and warnings before publishing[3].

### Format Code

Ensure consistent formatting:

dart format .

### Dry Run Publication

Test the publication process without actually publishing[3]:

dart pub publish --dry-run

This command checks:
- Package structure correctness
- pubspec.yaml validity
- Documentation completeness
- File permissions
- Scoring criteria for pub.dev

**Review the output carefully.** Fix any warnings or errors. The pub.dev scoring system evaluates[2]:
- Documentation quality (README, API docs, example)
- Code health (no errors, good test coverage)
- Platform support
- Popularity and maintenance

## Step 9: Publish to pub.dev

### First-Time Authentication

If this is your first package, authenticate with Google[3]:

dart pub publish

This will:
1. Open a browser for Google account authentication
2. Request permission to publish packages
3. Store credentials locally (one-time setup)

### Publish the Package

After authentication (or if already authenticated):

dart pub publish

Review the publication details and confirm when prompted.

**Important notes:**
- Once published, versions cannot be deleted (only marked as discontinued)[2]
- You can publish new versions anytime
- Package name reservations are permanent
- Consider the name carefully before publishing

### Verify Publication

After successful publication:
1. Visit `https://pub.dev/packages/gradient_text`
2. Verify metadata displays correctly
3. Check that documentation renders properly
4. Test the example in DartPad if applicable

## Step 10: Maintain Your Package

### Version Updates

When making changes, follow semantic versioning[2]:

\begin{itemize}
\item \textbf{MAJOR} (1.0.0 → 2.0.0): Breaking changes
\item \textbf{MINOR} (1.0.0 → 1.1.0): New features, backward compatible
\item \textbf{PATCH} (1.0.0 → 1.0.1): Bug fixes, backward compatible
\end{itemize}

To publish an update:

1. Make your changes
2. Update version in `pubspec.yaml`
3. Update `CHANGELOG.md`
4. Run tests: `flutter test`
5. Run analyzer: `flutter analyze`
6. Commit changes to git
7. Publish: `dart pub publish`

### Managing Uploaders

Grant publish access to other developers:

dart pub uploader add email@example.com
dart pub uploader remove email@example.com

### Package Metrics

Monitor your package performance on pub.dev:
- Download statistics
- Pub points score (max 130 points)
- Popularity metric
- Likes from users

## Common Issues and Solutions

### Issue: "Package validation failed"

**Solution:** Run `dart pub publish --dry-run` and fix all reported issues[3]. Common causes:
- Missing or incomplete README.md
- Invalid pubspec.yaml syntax
- Missing LICENSE file
- Insufficient API documentation

### Issue: "Version already exists"

**Solution:** Increment the version number in `pubspec.yaml`. Each publication requires a unique version[2].

### Issue: "Package name already taken"

**Solution:** Choose a different package name. Check availability at pub.dev before creating your package.

### Issue: Import errors in example app

**Solution:** Verify the example's `pubspec.yaml` correctly references the package using `path: ../`

### Issue: Low pub points score

**Solution:** Improve:
- Documentation coverage (add dartdoc comments to all public APIs)
- Example completeness
- Test coverage
- Platform support declarations
- Follow Flutter best practices

## Best Practices

\begin{enumerate}
\item \textbf{Single Responsibility}: Each package should solve one problem well
\item \textbf{Comprehensive Examples}: Provide multiple usage examples covering common scenarios
\item \textbf{Semantic Versioning}: Always follow semver rules to avoid breaking user code
\item \textbf{Changelog Updates}: Document all changes clearly for each version
\item \textbf{Responsive Maintenance}: Address issues and pull requests promptly
\item \textbf{Test Coverage}: Aim for high test coverage (>80\%) of your package code
\item \textbf{Platform Support}: Declare supported platforms explicitly in README
\item \textbf{Breaking Changes}: Provide migration guides when releasing major versions
\item \textbf{Dependencies}: Minimize external dependencies to reduce conflicts
\item \textbf{Documentation}: Write clear API docs with examples for all public members
\end{enumerate}
